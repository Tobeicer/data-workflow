# 游艺圈数据工作流

本工作区用于建设游艺圈数据资产生产线：接入数据源、保存原始证据、完成清洗治理、生成 L3 交付物。

当前不负责小程序、APP、Web 后台、支付、订单、交易功能和正式业务数据库建设。

## 目录

| 路径 | 用途 |
|---|---|
| `docs/` | 执行基线、分类参考和历史需求 |
| `data-workflow/` | 来源适配、采集脚本、运行产物和交付物 |
| `database/` | 数据库快照参考 |
| `AGENTS.md` | 代理执行人的边界与最小阅读规则 |

## 默认阅读顺序

普通数据任务只读：

1. `docs/数据工作流与游艺圈系统对接执行基线.md`
2. 当前来源的 `data-workflow/<source>/` 来源指南

`docs/project-split/` 是受保护的原始需求，只读不修改。`docs/requirements/信息整理.md` 是后续新增要求的唯一入口。

## 执行原则

- L0 原始资产不可覆盖；L3 不能替代 L0-L2。
- 大平台通过爬虫全量采集；行业店铺通过邀请入驻获取数据 API。
- 数据工作流不直接写正式业务表。
- 已确定的方案直接写现行做法，不在执行文档保留被放弃的选项。
