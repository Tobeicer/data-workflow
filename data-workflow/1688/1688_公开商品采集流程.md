# 1688 补充采集与公司资产流程

状态：商品、SKU、公司解析和单商品浏览器试采器已完成；待扩大到去重后的多公司队列。  
上位基线：`../../docs/数据工作流与游艺圈系统对接执行基线.md`

## 1. 定位与边界

1688 用于新品、参数、SKU、批发价、供应商和公司线索补充，不作为游艺圈正式商品主库的全量来源。

允许低频采集普通登录用户可见的商品、店铺、公司、工厂和主体资质信息。登录失效、验证码、滑块、403/429、权限或签名限制时停止，不绕过风控，不采集订单、购物车、聊天、会员和后台数据。

当前流程依次执行搜索列表、游艺相关性筛选、商品详情和 SKU 补采。正式批次应输出到 `data-workflow/runtime/runs/1688/<run_id>/`；旧脚本当前仍可能使用来源目录，迁移时以运行契约为准。

## 4. 公司资产

公司、店铺和商品独立建模：

- `source_product`：商品和 SKU；
- `source_shop`：店铺、经营信号和公开经营联系方式；
- `source_company`：工商主体；
- `source_factory_profile`：厂房、人员、设备、证书、专利和服务快照；
- 商品—店铺、商品—公司、店铺—公司关系。

关系必须保留来源页面、采集时间、匹配方法、置信度和冲突原因。店铺名、供应商名和证照主体名不得互相覆盖。

## 6. 质量与停止条件

- 商品、SKU、店铺和公司记录均有来源 URL 与采集时间。
- 未知数值使用空值，不写 0。
- 摘要数量与可枚举明细口径不同则分别保存。
- 专利/证书接口错误不得覆盖页面摘要值。
- 页面结构变化保存原始 HTML 并标记 `parser_drift`。
- 登录、验证码、限流或访问限制产生明确状态，不静默输出空成功。
