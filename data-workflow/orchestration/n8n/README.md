# n8n 控制面

状态：目录契约已建立；七个来源全部禁用，当前没有可标为 `active` 的工作流 JSON。

## 职责边界

n8n 只负责触发、来源锁、编排、重试、运行状态、人工门禁和告警。采集、清洗、图片处理、变化比较、质量检查和交付生成由 Python/Node 执行面完成，并通过机器可读的 `run_result.json` 返回结果。

更新策略最终支持 `scheduled`、`automatic` 和 `conditional` 三种触发模式；具体来源只能按其公开或授权边界、稳定性和运行证据选择模式。

## 启用门禁

来源只有同时通过以下四道门禁，才能把 `source_registry.json` 中的 `enabled` 改为 `true`：

1. JSON 实现：存在经过审查的来源工作流导出。
2. 凭据配置：凭据只配置在受控环境，不进入仓库或工作流导出。
3. dry-run：最小脱敏样本和状态路由验证通过。
4. 质量证据：质量门禁、失败状态和人工复核路径均有可核对证据。

缺少任一道门禁时，来源必须保持禁用，不得在 n8n 中标为 `active`。

## 安全与资产边界

禁止向 Git 提交密钥、Cookie、账号密码、浏览器 profile、原始大数据、图片库或运行产物。敏感配置使用本地未跟踪环境文件或 n8n 凭据存储；L0-L2 资产进入忽略的 `data-workflow/runtime/`，L3 交付进入忽略的 `data-workflow/deliveries/`。

工作流导出统一命名为 `<scope>_<source>_<version>.json`。导出前必须清除凭据引用中的敏感值，不得把不存在或未验收的工作流写成已启用状态。
