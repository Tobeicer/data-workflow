# 淘宝补充采集验证

状态：人工登录态搜索与详情参数补采原型已完成。  
上位基线：`../../docs/数据工作流与游艺圈系统对接执行基线.md`

## 1. 定位与边界

淘宝用于新品、价格区间、关键词、参数和同款候选补充，不作为正式商品主库的全量来源。

只进行人工授权、低频、小样本采集。登录、验证码、滑块、权限、签名或风控出现时停止，不采集订单、购物车、会员、地址、聊天和后台数据。

## 2. 当前文件

- 脚本：`collect_taobao_full_workflow.py`
- 测试：`test_collect_taobao_full_workflow.py`
- 当前结果：`taobao_product_category_full_20260709.csv`

## 3. 执行

登录准备：

```powershell
.\.venv-data\Scripts\python.exe data-workflow/taobao/collect_taobao_full_workflow.py --prepare-login
```

低频采集：

```powershell
.\.venv-data\Scripts\python.exe data-workflow/taobao/collect_taobao_full_workflow.py `
  --limit-per-keyword 2 `
  --output data-workflow/taobao/taobao_product_category_full_20260709.csv
```

流程为：搜索页商品卡片→详情参数→内存合并→单一结果 CSV。

## 4. 已验证结果

2026-07-09 使用 31 个游艺配件关键词，每个关键词最多 2 条：

- 搜索结果 62 行，唯一商品 61 个；
- 详情补采 61 条且全部成功；
- 型号 57 条、品牌 58 条、颜色/SKU 文本 52 条；
- 设备种类 41 条、控制系统 22 条。

淘宝详情参数完整度取决于商家填写情况，只能作为补全候选，不能覆盖合作商或授权来源中的高可信字段。

## 5. 交付规则

- 保存标题、商品 ID、价格、店铺、链接、图片、公开参数和采集时间。
- 结果进入 L0-L2 候选、快照或同款判断，不直接进入正式产品库。
- 登录 profile 和 Cookie 不进入 Git、文档或交付包。
- 解析为 0、登录失效、验证码或页面结构变化时返回明确状态并停止扩大请求。
