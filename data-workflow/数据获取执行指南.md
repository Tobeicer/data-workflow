# 游艺圈数据获取执行指南

版本：V1.1  
日期：2026-07-13

本文只说明日常数据获取与交付方法。职责、L0-L3、正式写库和当前优先级以 `../docs/数据工作流与游艺圈系统对接执行基线.md` 为准。

## 1. 来源优先级

| 优先级 | 来源 | 用途 |
|---|---|---|
| P0 | 授权 Excel、后台导出、官方 API、合作方文件 | 正式主数据优先来源 |
| P1 | 厂家官网、公开网页和商品页 | 长期公开补充来源 |
| P2 | 普通用户可访问的小程序接口 | 验证公开范围后接入 |
| P3 | 抓包识别公开接口 | 用于接口发现和验证，不作为唯一依赖 |
| P4 | PDF、画册、图片和 OCR | 字段补充，必须复核 |
| P5 | 大平台和潜在线索 | 低频候选，不直接进入正式库 |

1688、淘宝、京东、拼多多、抖音等只用于新品、参数、规格、价格、关键词、同款和商机线索，不追求全量。

## 2. 单来源标准流程

```text
登记来源与用途
→ 确认公开/授权边界和停止条件
→ 小样本验证
→ 记录字段和接口
→ 批量采集或文件接收
→ L0 原始归档
→ L1 标准化
→ L2 去重、关系、快照、变化和质量
→ AI 辅助与人工复核
→ L3 交付
→ 更新计划和健康检查
```

样本验证只用于确认字段、分页、唯一键、登录、风控和成本。已完成验证的来源不得在文档中继续保留为“待验证下一步”。

## 3. 运行前检查

- 来源用途、公开/授权属性和风险等级已登记。
- 登录态、配额、磁盘、输出目录和批次号可用。
- 请求频率、重试上限和停止条件明确。
- 有稳定来源 ID、组合键或内容哈希。
- 有最小测试样本和 dry-run。
- 不涉及订单、购物车、支付、地址、聊天、后台或非公开个人信息。

## 4. 每次运行产物

至少保存：

| 产物 | 内容 |
|---|---|
| L0 | 原始响应、文件、图片、请求元数据、来源证据和哈希 |
| L1 | 完整来源字段的标准化实体和明细 |
| L2 | 关系、快照、变化、质量、冲突和复核队列 |
| L3 | 当前平台契约需要的 XLSX/CSV/JSON/API 请求体 |
| 状态 | `run_result.json`、清单、错误记录和质量报告 |

价格、库存、销量、供货状态、文审状态和商机有效性必须带采集时间并保留历史。

## 5. 字段与关系原则

- 先保存来源原值，再生成标准值。
- 不用空字符串或 0 伪造未知信息。
- 公司、店铺、商品、SKU、图片、分类和文档分别建模，通过关系关联。
- 同一产品可有多个来源和供应商；同一配件可适配多个机型。
- 平台当前不接收的字段仍保留在 L0-L2。
- 分类、标签、实体合并和高风险结论保留置信度与复核状态。

## 6. AI 与人工复核

确定性任务优先使用规则：ID、哈希、价格、时间、URL、单位、格式和库存解析。

AI 可用于非结构化抽取、分类候选、相似判断、图片描述、缺失和复核原因。AI 输出必须结构化并带置信度；工商主体、认证、文审、侵权和低置信度合并必须人工确认。

## 7. 质量检查

每批至少检查：

- 总数、成功数、唯一数和重复率；
- 必填、来源、时间、图片和分类完整率；
- 新增、修改、下架、失败和无变化数量；
- 原始记录与清洗记录是否可核对；
- 页面结构、字段集合和枚举是否变化；
- 低置信度、冲突和复核数量。

数量异常下降、解析为 0、图片失败激增、价格批量归零、字段结构变化或预算超限时停止正常交付。

## 8. 停止与降级

遇到验证码、滑块、403/429、登录失效、权限不足、签名破解要求、来源异常或非公开数据时停止当前路径。

可降级为：人工登录、人工任务、小样本、文件导入、只保存页面可见信息或暂停来源。不得为了完成数量绕过平台限制。

## 9. 更新与交付

来源需记录最近成功时间、建议频率、结构变化、图片失效、新增、下架和人工处理状态。

交付前确认：

- 批次、来源、唯一键和字段字典明确；
- 质量报告、变化摘要和复核队列齐全；
- 图片和文档引用可访问；
- L3 可从 L0-L2 重建；
- 平台接收方式和错误回执已确认。

具体命令和来源限制只写在 `data-workflow/<source>/` 当前执行文档中。
