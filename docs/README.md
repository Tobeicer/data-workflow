# 游艺圈项目文档索引

`docs/` 只保存项目级基线、总体架构、治理和稳定参考。具体采集命令、来源结果和数据文件统一放在 `data-workflow/`。

## 当前文档

| 文档 | 唯一职责 |
|---|---|
| `数据工作流与游艺圈系统对接执行基线.md` | 当前权威：职责、L0-L3、写库边界、对接顺序和当前执行重点 |
| `游艺圈数据资产生产工作流总体执行方案.md` | 数据生产线的控制面、执行面、状态、质量和长期运行设计 |
| `游艺圈游戏游艺设备完整分类清单.md` | 完整设备的候选分类参考，不是平台正式枚举 |

平台当前 Excel 消费兼容格式位于根目录 `游艺圈数据导入字段规范_v2.md`。来源执行说明位于 `data-workflow/<source>/`。

## 文档优先级

1. 用户最新明确要求。
2. `数据工作流与游艺圈系统对接执行基线.md`。
3. 当前来源执行文档。
4. 总体方案或通用指南。
5. L3 消费兼容规范。
6. 历史需求与项目拆分。

同一规则只在最高优先级文档完整定义，其他文档使用引用，避免重复维护。

## 历史参考

- `project-split/`
- `requirements/`

这两个目录是受保护的早期需求和项目规划。仅在需要理解历史产品背景时读取；其中的旧职责、开发顺序、营销方式、数据路线和链接可能已失效。禁止修改，也不得用于覆盖当前执行基线。

## 维护规则

- 当前文档只写已确认方案，不保留 A/B/C 比较。
- 删除过期的“下一步”、等待状态和已完成试采步骤。
- 批次结果只保留日期、数量、结论和有效产物。
- 不记录命令失败、编码问题、重试过程或聊天推理。
- 新的职责、分层、数据库或系统对接决定先更新执行基线。
