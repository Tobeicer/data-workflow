## 大平台数据边界

游艺圈不是全网同类商品搬运库，也不以存储 1688、淘宝、京东、拼多多、抖音等大平台的全量商品为目标。

数据主线应以合作商、厂家、配件供应商和授权来源为主，例如漫立方、已合作配件商、厂家上传资料、供应商后台导出和人工审核资料。

1688、淘宝、京东、拼多多、抖音等大平台只作为补充来源：

1. 发现新品、新叫法、新规格和热门趋势。
2. 补充合作商缺失的参数、图片、规格、价格区间和关键词。
3. 辅助判断同款、相似款、竞品和市场行情。
4. 发现二手、促销、低价出售、求购等商机线索。

这些大平台不建议做复杂 API 接入，不追求全量商品数据，也不把爬取结果直接写入正式产品库。采集结果应先进入来源线索或候选资料，经过去重、可信度判断和人工复核后，再用于补全合作商品或生成商机。

## 产品数据完整性共识

`docs/project-split/04_数据采集与治理方案.md` 是早期原始需求参考，其中产品字段描述偏粗，不代表后续产品数据只需要名称、分类、图片等少量字段。

游艺圈的数据核心是可搜索、可比较、可询价、可补全、可追溯的产品资产。无论数据来自合作厂家、漫立方、配件供应商、授权表格，还是来自 1688、淘宝、京东、拼多多、抖音等公开平台，只要它描述的是一个商品，就都应按产品数据来采集和治理。

来源不同的区别：

1. 合作商/厂家/授权来源：可以作为正式产品主数据的重要来源，适合长期维护、深度补全、支持搜索、询价、库存、报价和履约评分。
2. 大平台公开来源：不做全量库，不直接成为正式商品；但应尽量保留公开可见的完整商品信息，用作新品发现、缺失字段补充、价格区间参考、同款/相似款判断和商机线索。

产品数据应尽量包含：

- 商品名称
- 分类
- 品牌
- 型号
- 厂家、店铺或供应商
- 来源平台、来源链接、来源商品 ID
- SKU、规格项、规格值、颜色、套餐、SKU 图片
- 价格、价格区间、批发阶梯价、促销价
- 库存、供货状态、销量或热度
- 图片、详情图、视频、说明书、画册
- 参数信息，例如控制系统、支付方式、主题风格、适用场景、适用年龄段、设备种类、尺寸、材质、电压、功率、接口、功能说明
- 适配机型
- 发货地、物流、交期
- 售后规则、质保、供应商信用
- 采集时间、更新时间、更新记录、可信度、复核状态

价格、库存、销量、供货状态属于会变化的快照型信息，不应只覆盖保存一个固定值。后续入库设计应保留采集时间和历史变化，便于判断价格走势、库存变化和来源可靠性。

SKU 不应被简化成一个文本字段。一个产品可以有多个 SKU，每个 SKU 可能对应不同规格、颜色、套餐、价格、库存和图片。采集时即使暂时无法完全标准化，也应尽量保留外部平台原始 SKU 结构和规格文本，后续再清洗映射。

## 潜在客户资料

可以通过企查查/天眼查/爱企查 企业付 API/批量导出(综合最优，最简落地)

企查查开放接口 `https://openapi.qcc.com/dataApi/886`，企业信息查询约 0.1 元一条，但不包含电话号码。企查查可以作为企业工商主体最权威的信息来源。

再用高德地图、百度地图、抖音、美团进行多方大数据信息校验，最终得到挖掘出来的精准信息。

### 操作流程（有开放api）

1. 开通企业商业版/API套餐(个人 VIP 仅支持少量导出，无法全国筛选)
2. 筛选行业关键词(经营范围包含:电玩、动漫游戏、游乐、儿童乐园、游艺娱乐、电子游艺);
3. 地域勾选全国，过滤「在营/存续」企业，一键批量导出 Excel;
4. 字段包含:企业名称、统一社会信用代码、法人、注册地址、经营范围、联系电话、经营状态、注册资本。
5. 企查查先确认企业主体是否真实、在营、经营范围是否匹配;
6. 高德/百度确认地址、门店、公开电话、营业状态;
7. 抖音/美团确认是否真实经营、是否活跃、是否有公开联系方式;
8. 多来源一致的企业进入高精准潜在客户池，不一致的进入人工复核。
