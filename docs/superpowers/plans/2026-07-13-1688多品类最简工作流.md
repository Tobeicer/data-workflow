# 1688 多品类最简工作流 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立 1688 五商品批量采集流程，完整保存商品/SKU，并按 `memberId` 去重采集店铺、公司、工厂和官方主体资质。

**Architecture:** 浏览器层只负责导航、页面结构化提取和 L0 证据；纯解析器生成商品/SKU；批量编排器先处理商品，再建立唯一公司队列，并输出统一 L1/L2 与批次状态。现有单商品公司解析继续复用，不引入数据库和其他平台。

**Tech Stack:** Python 3.11、Playwright、pytest、JSON/JSONL、现有 1688 持久化浏览器 profile。

- [ ] **Step 3: 实现批量流程**

接口：

```python
def run_multi_product_workflow(
    *, offers: list[dict], output_dir: Path, browser: BrowserSession,
    collected_at: str | None = None
) -> dict: ...
```

执行顺序：

1. 逐商品捕获商品页并保存 `l0/products/<offer_id>/`；
2. 用 `normalize_product_capture()` 生成商品和 SKU；
3. 从 `sellerModel` 提取 `memberId` 和店铺；
4. 写入 `checkpoint.json`；
5. 按 `memberId` 生成唯一公司任务；
6. 每家公司只捕获 shop、credit_detail、contact_info、business_info 四类页面；
7. 复用 `parse_company_asset()`；
8. 输出统一 L1/L2、质量报告、清单和运行结果。

- [ ] **Step 4: 实现失败隔离**

单商品或单公司失败写入对应状态和复核队列，继续执行后续任务；登录或验证码停止扩大线上请求，但保留已完成结果。
