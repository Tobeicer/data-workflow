# 数据工作流目录迁移收尾设计

日期：2026-07-15

状态：已批准实施

## 1. 目标

本轮只完成数据工作流目录迁移收尾，使代码、文档、本地资产和真实运行路径一致，为后续建设 n8n shared 控制面和各平台全量爬虫提供稳定基础。

本轮不实现或启用 n8n workflow，不启动在线全量采集，不设计平台字段渲染，不调整分类体系，不写游艺圈正式业务表。

## 2. 执行阶段

1. 关闭隔离分支复审遗留并验证。
2. 使用 `--ff-only` 快进主工作区，保留未跟踪交接文档。
3. 逐项校验并同盘移动漫立方、1688、淘宝和历史本地资产。
4. 先更新唯一执行基线，再同步 README、adapter、兼容指南和迁移映射。
5. 清理旧路径并完成全量测试、离线 dry-run 和最终资产验收。

## 3. 保护规则

- 永久保护 `docs/project-split/`；
- `docs/requirements/` 中只有 `信息整理.md` 可以接收新增确认需求；
- 禁止强制重置、强制 checkout 或覆盖用户成果；
- 大型资产和浏览器 profile 只做同盘移动，不复制、不合并；
- 目标已存在、资产清单不一致或路径越界时立即停止；
- 所有来源在完整门禁通过前保持 `enabled=false`。

## 4. 完成定义

- 复审遗留关闭且治理测试可移植；
- 主工作区安全快进到迁移分支；
- 本地资产按清单完成可证明的同盘移动；
- 活跃文档、目录、命令和真实资产位置一致；
- 全部最终验收通过；
- n8n registry 仍保持全禁用；
- 未开始在线爬虫或 n8n workflow 实现。
