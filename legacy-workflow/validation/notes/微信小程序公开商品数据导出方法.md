# 微信小程序公开商品数据验证指南

版本：V1.0  
定位：新小程序来源的接口发现与可行性验证方法，不是当前执行计划。  
已验证样板：漫立方；具体命令见 `../manlifang/漫立方抓包流程.md`。

## 1. 适用范围

适用于普通微信用户正常浏览可见的分类、商品列表、详情、SKU、价格、库存、图片和参数。

不适用于订单、购物车、会员、支付、地址、聊天、后台接口，以及需要绕过登录、权限、签名或风控的数据。

核心原则：普通用户无法自然访问的内容不采；需要破解或绕过时停止该路径。

## 2. 可行性条件

进入脚本开发前确认：

- 手机可通过代理正常访问小程序；
- 抓包工具能看到目标公开请求和 JSON；
- 分类、列表、详情接口可由正常会话复现；
- 分页字段清晰；
- 有稳定商品 ID 或可解释的去重键；
- 请求不依赖后台权限或破解签名。

只能看到域名、接口持续返回受限状态、请求属于用户/交易数据或必须破解签名时，判定当前路径不可自动化。

## 3. 环境准备

推荐使用 mitmproxy/mitmweb；Fiddler 或 Charles 可用于人工交叉验证。

基本步骤：

1. 手机与电脑连接同一局域网。
2. 启动 `mitmweb --listen-port 8888`。
3. 手机 Wi-Fi 代理指向电脑 IP 和端口 8888。
4. 打开 `http://mitm.it` 安装并信任证书。
5. 用普通 HTTPS 页面验证代理，再打开目标小程序。

证书、代理或网络未验证前，不进入接口分析。

## 4. 接口发现顺序

```text
分类树
→ 商品列表与分页
→ 商品详情
→ SKU/规格与价格库存
→ 图片和详情内容
```

操作时记录“页面动作→请求”的对应关系：打开首页、进入分类、滚动分页、搜索、打开详情、切换规格和加载图片。

优先关注路径或字段：

```text
category / product / goods / list / detail / search / sku / spec / stock
id / parentId / page / pageSize / offset / records / items / images
```

看到 `cart`、`order`、`member`、`user`、`address`、`payment`、`admin`、`backend` 等路径时先停止并确认是否属于非目标数据。

## 5. 接口记录模板

每个关键接口保存：

```text
接口类型：分类 / 列表 / 详情 / SKU / 图片
触发动作：
请求 URL 和方法：
请求参数或请求体：
普通访问自然产生的必要 headers：
返回 JSON 样本：
分页字段：
商品 ID 字段：
来源会话要求：
公开/授权判断：
停止条件：
```

同时保存原始响应、请求时间、页面 URL、必要截图和文件哈希。Cookie、token 和个人登录态不得写入文档或样本库。

## 6. 最小验证

### 分类

- 可取得分类 ID、名称和父子关系。
- 能识别叶子分类或遍历方式。

### 列表

- 可按分类或搜索返回商品数组。
- 能控制分页并取得商品 ID。
- 第 1 页和第 2 页请求差异明确。

### 详情与 SKU

- 商品 ID 能关联详情。
- 详情图、参数、规格、SKU、价格或库存可追溯。
- 切换规格产生的组合来自接口，不自行生成不存在的 SKU。

### 脚本复现

- 使用复制的请求或最小脚本能得到相同结构。
- 不依赖后台权限、破解签名或扩大请求频率。

样本数量以覆盖字段和边界为准，不设置固定数量门槛。

## 7. 正式接入条件

新来源进入全量或周期任务前必须具备：

- 来源用途、授权/公开属性、频率和停止条件；
- 分类、列表、详情和 SKU 的实际可用范围；
- 唯一键、分页、去重和断点方式；
- L0 原始归档和最小固定样本测试；
- 字段映射、质量指标、异常状态和复核队列；
- 图片保存、哈希和失效处理；
- L3 交付用途和更新频率。

## 8. 图片与文件

- 先保存原始 URL 和来源响应。
- 抽样验证访问、格式、尺寸和临时签名。
- 需要长期交付时下载原图、计算哈希并转存自有存储。
- 记录商品 ID、图片角色、原始 URL、本地路径、下载时间、状态和失败原因。

## 9. 停止与降级

遇到以下情况停止：

- 需要破解签名、绕过登录或访问后台；
- 请求涉及用户、订单、支付、地址或聊天；
- 频繁风控、封禁、403/429 或服务异常；
- 页面结构无法稳定识别且没有可测试样本。

可降级为人工样本、页面可见信息整理、授权文件接收或暂停来源。不得用高频请求弥补接口不稳定。

## 10. 验证交付清单

```text
来源名称与入口
公开/授权边界
分类、列表、详情、SKU 请求样本
每类最小原始响应
字段映射和唯一键
分页、去重、频率和停止条件
图片策略
可脚本复现结论
质量风险与下一步
```

验证通过后，为该来源建立独立 `data-workflow/<source>/` 执行文档和适配器；本通用指南不保存来源专属命令或历史批次日志。
