- 不修改 `docs/project-split/`、`docs/requirements/` 及其内容。
- 不删除当前正式批次的原始响应、结构化 JSONL、哈希命名原图、批次元数据和清洗主数据。

- 删除前记录候选路径、大小、生成来源、引用状态和可重建依据。

## 已知环境限制

- PowerShell 执行策略阻止 `codegraph.ps1`，后续统一使用 `E:\nodejs\node_global\codegraph.cmd`。
- 当前 PowerShell 不支持 `??` 空值合并语法，统计脚本使用兼容写法。
- PowerShell 向原生 Python 传递含双引号的多行 `-c` 参数时丢失内部引号；XLSX 审计改用单引号 Python 字面量。
- 旧版 PowerShell 不接受未包裹的 `foreach (...) { ... } |`；批量结果先保存到数组再输出。
- 旧版 PowerShell `Remove-Item` 删除 `runtime/tmp` 内目录符号链接时触发 `NullReferenceException`，本次在其他删除前中止；改用已验证目标边界后的 `.NET Directory.Delete` 删除链接。
- 首次数据完整性断言把 8499 条成功下载关系误写为 5527 个唯一图片数；哈希与文件均通过，修正关系行/唯一实体口径后重跑。
